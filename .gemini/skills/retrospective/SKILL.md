# Retrospective (振り返り) プロトコル

タスクの完了時やセッションの区切りに本スキルを起動し、以下の構成で振り返りを行ってください。

## 1. 実施内容 (What was done)
- セッション内で完了した主要な作業を箇条書きで記述。

## 2. 学び・知見 (Learnings)
- 調査や作業を通じて得られた新しい知見、プロジェクト固有の仕様、技術的な発見を記述。

## 3. 発生した問題点 (Problems)
- ツール実行エラー、環境制約による困難、技術的なハマりどころなどを記述。

## 4. コミュニケーションの反省 (Communication Failures)
- ユーザーの意図を誤解した点、独断で進めてしまった点（「抱き合わせ実装」など）、意図の不一致が指摘された箇所を正直に記述。

## 5. 人間側へのアドバイス (Advice for Human)
- **今回のセッションでの具体的な出来事や失敗、成功体験に基づき**、ユーザー側がどのように指示を出し、どのように振る舞えば、より安全（ユーザーのコントロール権が維持され、推測ではなく観測に基づいた確実な進捗が得られる状態）かつ効率的に目的を達成できたかのアドバイスを記述。

## 6. 改善提案 (Improvement Ideas)
- 将来のミスを防ぎ、作業品質を向上させるための `GEMINI.md` や各スキルの具体的な修正案を記述すること。
- **「気をつけます」「習慣にします」といった精神論は厳禁とする。** 改善案は必ず成果物（ファイル）の更新案として、そのまま適用可能な形で提示すること。

## 振り返りの目的
AIエージェントが自らの行動を客観的に評価し、プロジェクトの「標準作業手順（SOP）」（GEMINI.md や AIエージェントスキル）を継続的にブラッシュアップすることで、将来の作業効率と対話の正確性を高める。
